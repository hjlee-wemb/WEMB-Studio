UI 수정사항
- 툴팁 마우스 오버하면 무조건 마우스 왼쪽으로 나오고 있는데 마우스 오른쪽으로 나오게 수정하기 (왼쪽 메뉴이다 보니 짤림)
- 되돌리기 노출이 약함, 타임라인/단축키 UI 명확하게 할 것
- 전체 기본값으로 돌리기 ui 필요 
- 툴 크롬 ↔ 프리뷰 테마 시각 분리: 사이드바/모달은 중립 UI 팔레트 고정, 프리뷰만 생성 테마 적용(경계선/배경 톤으로 구분).
  툴 쪽은 모노톤으로 구분하기
- 엔터프라이즈 부분에서 헤더 구분해놓은게 삐꾸나는데 확인하기

UX 수정사항






검증이 "숨어 있음": 대비 상태를 보려면 모달을 열어야 함. 현재 팔레트의 AA/AAA 통과 여부가 상시 보이지 않음.
되돌리기(undo/redo) 노출 약함: scheduleHistory로 히스토리는 쌓지만 타임라인/단축키 UI가 명확치 않음.
성능·모션 리스크: liveDot·okPulse·alarmBlink·critPulse·csSpin·csIndeterminate·스켈레톤 등 상시 애니메이션이 많은데 prefers-reduced-motion은 딱 3곳(valBump·배경 드리프트 등)만 적용됨. 실제로 이번 세션에서 애니메이션이 렌더러 스타일 계산을 굶겨 스크린샷·조회가 지연되는 것을 목격 → 저사양/모바일에서 체감 저하 + 접근성 이슈.
