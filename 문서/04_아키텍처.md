## 4.1 아키텍처 개요

WEMB Studio는 **백엔드가 없는 단일 HTML 클라이언트 애플리케이션**이다. 모든 로직(테마 생성·검증·렌더·산출)이 브라우저에서 실행되며, 상태는 브라우저 저장소와 URL로 관리한다. 정적 호스팅(GitHub Pages)만으로 배포·운영된다.

```
┌─────────────────────────────────────────────────────────────┐
│                        Browser (Client)                      │
│                                                              │
│   index.html  ──────────────────────────────────────────    │
│   ├─ <style>  : 테마 토큰(CSS Variables) + 레이아웃/컴포넌트   │
│   └─ <script> : Vanilla JS 모듈                              │
│        ┌──────────────┬──────────────┬──────────────────┐    │
│        │ Theme Engine │ WCAG Engine  │ Dashboard Render │    │
│        │ (팔레트 생성) │ (대비 검증)   │ (차트/위젯)       │    │
│        ├──────────────┼──────────────┼──────────────────┤    │
│        │ Authoring    │ Theme Mgmt   │ Export           │    │
│        │ (배치/내용)   │ (스냅샷/AB)   │ (토큰/이미지/URL) │    │
│        └──────────────┴──────────────┴──────────────────┘    │
│                                                              │
│   State: localStorage  +  URL(직렬화 공유)                    │
└──────────────┬───────────────────────────────┬──────────────┘
               │ CDN                             │ External API
        Pretendard 폰트                     네이버 날씨(실시간)
```

## 4.2 아키텍처 원칙

| 원칙 | 설명 |
|---|---|
| No-Backend | 서버·DB 없음. 정적 파일만으로 완결 |
| Single-File | 배포·이식 단순화를 위해 단일 `index.html` 유지 |
| Token-Driven | CSS 변수 토큰이 단일 진실원(Single Source of Truth) — 값 교체로 전 화면 전환 |
| Framework-Free | 외부 프레임워크/빌드 도구 없이 Vanilla JS |
| Progressive Disclosure | 고급 옵션(미세조정 등)은 필요 시에만 노출 |

## 4.3 레이어 구성

### 4.3.1 프레젠테이션 레이어
- CSS 변수 기반 테마 토큰 + 반응형 레이아웃(2단, 100vh)
- 라이트/다크 · Flat/Enterprise 시각 체계를 토큰 스위칭으로 전환

### 4.3.2 도메인 로직 레이어 (JS 모듈군)
| 모듈 | 책임 | 대표 함수 |
|---|---|---|
| Theme Engine | 팔레트 생성·역할 매핑·미세조정 | `genPalette`, `computeHexMap`, `applyAdjust` |
| WCAG Engine | 대비비 계산·자동 보정 | `wcagRatio`, `enforceAAA`, `fixThemeContrast` |
| Enterprise | ≤50색 토큰 확장·고급 시각 | `genEnterprise`, `enhanceEnterpriseBars` |
| Dashboard Render | 차트/위젯 렌더·애니메이션 | `renderDonut`, `renderGauge`, `makeThroughput` |
| Authoring | 배치/내용/차트 종류 편집·추천 | `setLayout`, `setContent`, `recommend` |
| Theme Mgmt | 스냅샷·히스토리·A/B 비교 | `addSnapshot`, `commitHistory`, `renderAB` |
| Export | 토큰/이미지/공유 URL 산출 | `genCSS`, `genFigma`, `shareURL` |
| Realtime | 시계·날씨·KPI 갱신 | `fmtClock`, `loadWeather`, `liveKPI` |

### 4.3.3 상태·지속성 레이어
| 대상 | 메커니즘 | 수명 |
|---|---|---|
| 콘텐츠 차트/데이터 종류 | localStorage | 영구(브라우저) |
| 테마 스냅샷 | localStorage | 영구(브라우저) |
| 되돌리기 히스토리 | 인메모리 스택 | 세션 |
| 공유 시안 | URL 쿼리 직렬화 | 링크 단위 |

## 4.4 외부 의존성

| 의존 | 용도 | 방식 |
|---|---|---|
| Pretendard (jsDelivr CDN) | 한글 최적화 폰트 | `<link>` |
| Open-Meteo API | 실시간 날씨(기온·날씨코드) | `fetch()` — 무인증 |
| Browser Geolocation API | 위치 좌표(최초 1회, 서울 폴백) | `getCurrentPosition` |
| html2canvas-pro (jsDelivr CDN) | 화면 PNG 캡처 | 필요 시 동적 로드(`loadScript`) |

## 4.5 배포 아키텍처

```
개발(로컬 index.html)
      │  git push
      ▼
┌──────────────────────────┐     ┌──────────────────────────┐
│ origin: localee94/        │     │ hjlee: hjlee-wemb/        │
│ WEMB-studio               │     │ WEMB-Studio               │
└────────────┬─────────────┘     └────────────┬─────────────┘
             │  GitHub Pages 빌드              │
             ▼                                 ▼
        공개 정적 사이트 (HTTPS)          (미러 저장소)
```

- **이중 원격 저장소**: `origin`(localee94) + `hjlee`(hjlee-wemb)에 동시 push
- **CI/CD**: 별도 파이프라인 없이 GitHub Pages 정적 배포
- **롤백**: git 커밋 히스토리 기반 (Revert 사례 있음)

## 4.6 아키텍처 특성 및 트레이드오프

| 장점 | 트레이드오프 |
|---|---|
| 배포·이식 극도로 단순(정적 1파일) | 파일 크기 증가(단일 파일 8,687 lines) |
| 인프라 비용/운영 부담 없음 | 서버측 로직·인증·영속 저장 불가 |
| 프레임워크 종속성 없음 | 모듈 경계가 파일 내 관례에 의존 |
| 토큰 기반으로 테마 확장 용이 | 대규모 협업 시 파일 분리 리팩터링 필요 |

---

## 4.7 향후 확장 방향
- 관제 실데이터 연동(WebSocket/REST) — 현재 시뮬레이션 대체
- 산출 토큰의 디자인–개발 파이프라인 자동 연계(Figma 변수 동기화)
- 대규모화 대비 모듈 파일 분리 및 빌드 도입 검토
