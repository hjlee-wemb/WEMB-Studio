## 3.1 화면 설계

### 3.1.1 전체 레이아웃
- **2단 구성**: 좌측 **설정 패널**(접이식) + 우측 **관제 대시보드 콘텐츠**(반응형, 100vh)
- 상단: 브랜드 로고(테마별 교체) · 실시간 시계 · 날짜/날씨
- 폰트: Pretendard (CDN), 최소 12px

### 3.1.2 좌측 설정 패널 구성
| 그룹 | 세부 |
|---|---|
| MODE | 라이트/다크, Flat/Enterprise 버전 토글 |
| SEED COLOR | 기준 색 지정, 색 추출 방식(하위 메뉴), 배색 방식, 미세조정(더보기) |
| 이미지 첨부 → 테마 추출 | 아이콘형 업로드, 파일명/확장자 표기(small) |
| 추천 테마 | 올해의 트렌드 컬러, 다크/라이트 탭 구분 |
| 현재 팔레트 | 색상 칩(피커/텍스트 지정/복사/초기화), 접이식 |
| 대시보드 편집 | 배치 수정 / 내용 수정 / 디자인 편집(모서리 등) |

### 3.1.3 상호작용 규칙
- 활성 테마 외 소스는 **disable** 처리(현재 무엇이 활성인지 명확히 표시)
- **미세조정은 기준 색에서만** 가능하며 **우측 콘텐츠에만 반영**(좌측 메뉴 색 고정)
- 하위 메뉴는 hover 시 우측에 펼쳐지는 플라이아웃 방식
- 표 hover 피드백, 편집 모드 시 텍스트 커서 신호 제공

## 3.2 테마 토큰 설계 (Design Tokens)

CSS 변수 기반으로 테마를 추상화하여, 토큰 값 교체만으로 전체 UI가 일괄 전환된다.

| 토큰 | 역할 |
|---|---|
| `--bg-page` | 페이지 배경 |
| `--bg-surface` | 카드/서피스 배경 |
| `--bg-accent` | 강조 배경 |
| `--point-main` | 주 포인트 색 |
| `--point-sub` | 보조 포인트 색 |
| `--text-strong` | 강조 텍스트 |
| `--text-weak` | 약한 텍스트 |
| `--line` | 구분선/보더 |
| `--warning` / `--danger` | 경고/위험 상태색 |
| `--radius-panel` | 모서리 둥글기 |
| `--font` | 폰트 패밀리 |

> 색은 HSL 기반으로 관리되며, 명도(L) 조정으로 대비비 보정을 수행한다.

## 3.3 기능 모듈 설계

### 3.3.1 컬러 연산 (Color Math)
- `hexToRgb`, `rgbToHsl`, `hslToHex`, `hslToCss`, `clamp`
- 색 표현 변환 및 정규화(`normHex`)의 기반 유틸

### 3.3.2 팔레트 생성 (Palette Generation)
- `genPalette` — 기준 색 + 배색 방식으로 팔레트 산출
- `buildDirect` / `computeHexMap` / `adjustHex` — 역할별 색 매핑 산출
- `applyAdjust` / `resetAdjust` — 미세조정(H/S/L) 적용·초기화

### 3.3.3 접근성 검증 (WCAG Engine)
- `relLum`(상대휘도) → `contrast` / `wcagRatio`(대비비) 계산
- `enforceAAA` / `enforceContrast` / `fixContrastLightness` — 미달 색을 통과 수준으로 명도 보정
- `contrastRows` / `renderContrast` / `fixThemeContrast` — 검증표 렌더 및 일괄 보정

### 3.3.4 Enterprise 시각 체계
- `genEnterprise` / `entCoreMap` — ≤50색 토큰 확장 생성
- `enhanceEnterpriseBars` — 은은한 그라데이션 배경 애니메이션 등 고급 시각 처리

### 3.3.5 대시보드 렌더 (Charts / Widgets)
- 차트: `makeThroughput`(라인), `makeConnected`, `makeCircuit`, `makeDonut`, `makeHeat`
- 렌더러: `renderAxis`, `renderDonut`, `renderGauge`, `renderHeat`, `renderProgress`, `renderTable`, `genericRender`
- `spline` 등 곡선 보간, 진입 애니메이션(`intro`)

### 3.3.6 콘텐츠 저작 (Authoring)
- 배치: `applyLayout` / `setLayout`
- 내용/차트 종류: `setContent`, `saveTypes` / `loadTypes`, `recommend`(제목 기반 추천)
- 디자인: `applyRadius`, `syncVersionUI`
- 하위 메뉴: `openSub` / `hideSubs` / `openMenu` / `buildPicker`

### 3.3.7 실시간 요소
- `fmtClock`(시계), `loadWeather` / `initWeather`(날씨 — Open-Meteo API + 브라우저 Geolocation, 위치 거부 시 서울 폴백)
- `liveKPI` / `animateVal` / `tick` — KPI 카운트업 주기 갱신

### 3.3.8 테마 관리 (Snapshot / History / A/B)
- 히스토리: `captureTheme`, `restoreTheme`, `themeSig`, `commitHistory`, `renderHistory`, `jumpHistory`
- 스냅샷: `loadSnapshots` / `saveSnapshots` / `addSnapshot` / `deleteSnapshot` / `renderSnapshots`
- A/B 비교: `abThemeList`, `refreshABOptions`, `namespaceIds`, `renderThemeInto`, `renderAB`, `openAB`

### 3.3.9 산출 (Export)
- 토큰: `genCSS`, `genJSON`, `genFigma`, `genTailwind`
- 이미지: `exportPNG`, `genPaletteSVG`, `exportSVG`
- 공유: `encodeState` / `decodeState` / `shareURL` / `copyShare` / `applyShareFromURL`

### 3.3.10 컴포넌트 시트 (State Gallery)
- 화면 사용 컴포넌트 + 미노출 추정 컴포넌트/상태값 열람, 상태 히트맵, HTML 내보내기(`csExport`)

## 3.4 상태 관리 설계

| 상태 | 저장 위치 |
|---|---|
| 콘텐츠 차트/데이터 종류 | localStorage (`saveTypes`/`loadTypes`) |
