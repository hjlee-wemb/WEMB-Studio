## 1.1 프로젝트 개요

**WEMB Studio**는 컬러 테마를 생성·검증·관리하고, 그 결과를 **실시간 관제 대시보드**에 즉시 반영해 미리보기할 수 있는 **웹 기반 테마 저작(Authoring) 도구**이다.

관제 시스템 대시보드의 색상 체계를 **포인트 색상 변경만으로 전체 테마가 일관되게 전환**되도록 표준화하고, 색 전문 지식이 없는 사용자(고객사 운영자·기획자)도 접근성(WCAG) 기준을 충족하는 테마를 만들어 CSS/JSON/Figma/Tailwind 토큰으로 산출할 수 있도록 하는 것이 목적이다.

---

## 1.2 추진 배경

- 기존 관제 대시보드 디자인(Figma `system` 파일, key `1fIpWV8TozxhsKD5qIX35D`)은 색상이 화면 곳곳에 하드코딩되어, 테마 변경 시 수작업 비용이 컸다.
- 색상을 **디자인 토큰(변수)** 으로 추상화하여, 포인트 색 하나로 테마 전체를 제어할 수 있는 구조가 필요했다.
- 비개발 실무자도 **접근성 기준을 자동으로 만족**하는 테마를 스스로 구성·산출할 수 있는 셀프서비스 도구에 대한 요구가 있었다.
- 참고 서비스: BairesDev *AI Colors*(색 기반 테마 전환), Pantone *올해의 컬러 2026 (Cloud Dancer)*.

## 1.4 주요 기능 요약

1. **테마 생성** — 기준 색(Seed)·배색 방식(Harmony)·이미지 추출·랜덤·올해의 트렌드 컬러
2. **접근성 검증** — WCAG 대비비 자동 계산, AAA 필터링, 미달 자동 보정
3. **테마 관리** — 스냅샷 저장·버전 히스토리·되돌리기, A/B 비교 뷰
4. **산출(Export)** — CSS 변수 / JSON 토큰 / Figma 변수 / Tailwind config / PNG·SVG / 공유 URL
5. **대시보드 저작** — 카드 배치·내용·차트 종류·디자인(모서리 등) 편집
6. **시각 체계** — Flat(≤10색) / Enterprise(≤50색) 토글
7. **실시간 요소** — 실시간 시계·날씨, KPI 카운트업, 차트 애니메이션
8. **라이트/다크 모드** — 설정 패널·콘텐츠·팝업 통합 동기화

## 1.5 대상 사용자

| 사용자 | 활용 목적 |
|---|---|
| 고객사 운영자 | 관제 대시보드 테마를 직접 선택·적용하여 운영 환경에 반영 |
| 내부 기획자 / PM | 설계서 근거(대비비 표·토큰)로 테마를 정의하고 산출물 확보 |
| 개발자 | 산출된 CSS/JSON/Figma/Tailwind 토큰을 실제 구현에 연계 |

## 1.6 기대 효과

- 테마 변경 리드타임 단축(수작업 색상 교체 → 토큰 기반 일괄 전환)
- 접근성 기준(WCAG) 위반 리스크 사전 차단
- 디자인–개발 간 토큰 표준화로 **핸드오프 비용 절감**
- 고객사 셀프서비스로 커스터마이징 대응 부담 완화
