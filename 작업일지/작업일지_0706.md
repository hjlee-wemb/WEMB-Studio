## 개요

WEMB Studio(컬러 테마 스튜디오)의 **사용성·시각 일관성**을 전반적으로 개선함. 도움말 툴팁의 방향·잘림·겹침·정렬 문제를 구조적으로 해소하고, **툴 UI(크롬)와 생성 테마(프리뷰)를 시각적으로 분리**함. 되돌리기·전체 초기화 흐름을 정비하고, **Color Hunt 인기 팔레트 연동**과 사진 기반 테마 추출 로직을 개선함. 엔터프라이즈 헤더·표 렌더링 결함을 수정하고, 완성 화면을 **Figma로 핸드오프(코드→디자인)** 한 뒤 형상관리에 반영함.

## 핵심 기능 요약

- **가려지지 않는 도움말:** 툴팁을 오른쪽으로 펼치고 어떤 조상 레이어에도 갇히지 않게 처리해 잘림·겹침·오정렬 없이 항상 완전하게 노출
- **툴↔프리뷰 시각 분리:** 사이드바·모달은 항상 중립 모노톤으로 고정, 생성 테마는 오른쪽 프리뷰에만 적용해 편집 중 크롬 일관성 보장
- **단일 초기화 흐름:** 되돌리기 단축키·타임라인과 별개로, 클릭 한 번으로 전체 옵션을 초기 상태로 되돌리는 초기화 컨트롤 제공(저장 스냅샷은 유지)
- **Color Hunt 큐레이션 갤러리:** 인기 팔레트 40종을 다크/라이트 탭·페이지네이션으로 탐색, 사진 추출은 전체 색 기반으로 대표색 정확도 향상

## 기술 구성

- **구조:** 단일 HTML + CSS 변수 토큰 + Vanilla JS(외부 의존성 없음)
- **테마 격리:** `:root`를 고정 중립 모노톤으로 정의, 생성 팔레트는 `.main`(프리뷰)에만 인라인 적용 / 전역 `data-mode`는 다크 고정해 모달 프레임이 프리뷰 밝기를 따라가지 않도록 처리
- **툴팁 렌더링:** 표시 시 `document.body`로 이동 + `position:fixed` 좌표 보정으로 `overflow`·`filter`·`sticky` 컨테이닝 이슈 회피
- **추천 팔레트:** `PALETTES` 배열(Color Hunt 40종) 기반, 다크/라이트 자동 분류 + 페이지당 8개 페이지네이션
- **Figma 핸드오프:** Figma Plugin API(use_figma)로 `WEMB Studio — App` 프레임(1280×820) 구성, Pretendard 미설치 환경이라 `Gothic A1`로 대체

## 이슈 & 해결

- 툴팁이 사이드바 오버플로우·`filter`·`sticky` 조상에 갇혀 잘리거나 가려짐 → `body` 이동 + 뷰포트 충돌 보정으로 해소
- 엔터프라이즈 표의 밴드 헤더가 상자형 세로선으로 보이고 셀이 붙음 → 보더 지정을 하단으로 한정하고 배경을 `background-color`(padding-box 유지)로 교체해 간격 복원
- 사진 추출이 작은 원색 얼룩(가운데 피사체)에 치우침 → 전체를 면적 가중으로 표본화해 대표색 선정
- colorhunt.co 실시간 로딩 불가(CORS) → 인기 팔레트를 엄선해 내장하고 사이트 링크만 유지
- Figma 편집 권한 부재 → Full 시트 계정으로 전환해 쓰기 수행(픽셀 캡처 대신 Plugin API로 구성)
