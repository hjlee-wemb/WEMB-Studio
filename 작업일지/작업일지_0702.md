## 개요

사이드바 **테마 동기화**와 콘텐츠 **차트 저작**을 확장하고, 색상 규모 기준의 **시각 체계 이원화(Flat / Enterprise)** 를 도입함.

## 주요 작업 내역

| 구분        | 작업 내용                                                                                                | 상태 |
| ----------- | -------------------------------------------------------------------------------------------------------- | ---- |
| 테마 동기화 | 사이드바가 콘텐츠의 light/dark 모드를 동기화해 따라가도록 수정                                           | 완료 |
| 콘텐츠 저작 | '내용 수정'에 차트 종류 전환 추가 — 현재 사용 차트 전부 선택 가능, 제목 내용 기반 차트/데이터 추천       | 완료 |
| 추천 구조   | 데이터/차트 카테고리 구분, 카테고리 hover 시 우측 플라이아웃 서브메뉴                                    | 완료 |
| 버그 수정   | 위치 권한 반복 요청 제거 — 최초 1회만 요청, 실패 시 서울 좌표 폴백                                       | 완료 |
| 시각 체계   | Flat(≤10색) / Enterprise(≤50색) 버전 도입·토글, Enterprise 확장 토큰 + 은은한 그라데이션 배경 애니메이션 | 완료 |
| 정리        | 라이트 모드 그림자/그라데이션 과다 완화, 콘텐츠 차트 그림자 제거                                         | 완료 |
| 형상관리    | GitHub push(origin·hjlee)                                                                                | 완료 |

## 핵심 기능 요약

- **모드 일관성:** 사이드바–콘텐츠–팝업이 하나의 라이트/다크 모드로 동작
- **차트 저작:** 콘텐츠 성격에 맞는 차트를 추천·전환
- **버전 이원화:** 색상 규모에 따라 Flat/Enterprise 시각 체계 선택

## 기술 구성

- **구조:** 단일 HTML + CSS 변수 토큰 + Vanilla JS
- **Enterprise:** ≤50색 확장 토큰 생성기(`genEnterprise`) + 서피스 단계·그라데이션
- **위치/날씨:** Geolocation 최초 1회 + 서울 폴백

## 이슈 & 해결

- Flat/Enterprise 차이가 미미 → Enterprise 시각(그라데이션·표면 단계) 강화로 구분 명확화
- 위치 권한 팝업 반복 → `getCurrentPosition` 1회 호출·좌표 캐시로 전환
- Enterprise에서 미세조정·팔레트 추천 반영 누락 → 반영 로직 보완
