## 개요

Enterprise **미세조정 반영 범위**를 바로잡고, **라이트 모드 완성도**(배경 명도·팝업/모달·버튼 가시성)를 높임. **컴포넌트 시트**를 실제 화면 컴포넌트와 동일 기준으로 검증·정비하고, 고객사·PM 관점의 UX를 재검토함.

## 주요 작업 내역

| 구분        | 작업 내용                                                                                          | 상태 |
| ----------- | ------------------------------------------------------------------------------------------------- | ---- |
| 반영 범위   | Enterprise에서 기준색 미세조정 시 좌측 메뉴는 기준 팔레트 유지, 우측 콘텐츠에만 반영 (Flat/Enterprise 공통) | 완료 |
| 라이트 모드 | Enterprise 라이트 모드 배경 명도 상향(칙칙함 완화)                                                 | 완료 |
| 라이트 모드 | 팝업·모달을 라이트 모드에 맞는 디자인으로 전환(`html[data-mode]` 전역 동기화)                      | 완료 |
| 컴포넌트 시트 | 상태 히트맵 축소·대시보드 방식과 동일화, 추측 컴포넌트를 실제 콘텐츠 컴포넌트와 동일 기준으로 검증(명도 대비 포함) | 완료 |
| 레이아웃    | 라인/영역차트 크기 축소, 작은 항목끼리 묶어 3열 dense 레이아웃으로 화면 낭비 최소화                | 완료 |
| 버그 수정   | 라이트 모드 컴포넌트 시트 팝업의 HTML 내보내기 버튼이 흰색으로 묻히는 문제 수정                    | 완료 |
| UX 검토     | 고객사 운영자·내부 PM 버전 관점의 UX 평가 — 좋은 점·개선 필요 점·개선안 도출                       | 완료 |
| 형상관리    | GitHub push(origin·hjlee)                                                                          | 완료 |

## 핵심 기능 요약

- **일관된 미세조정:** 색 미세조정이 좌측 메뉴를 흔들지 않고 우측 콘텐츠에만 반영
- **라이트 모드 완성:** 배경 명도·팝업/모달·버튼까지 라이트 모드에서 일관되게 동작
- **신뢰할 수 있는 컴포넌트 시트:** 화면 컴포넌트와 동일 기준으로 검증된 상태 갤러리

## 기술 구성

- **구조:** 단일 HTML + CSS 변수 토큰 + Vanilla JS
- **모드 동기화:** `html[data-mode]` 기준으로 콘텐츠·사이드바·팝업·모달 일괄 전환
- **컴포넌트 시트:** 색면 글자 대비(on-color) 검증, dense 그리드 레이아웃

## 이슈 & 해결

- Enterprise 미세조정이 좌측 메뉴까지 바뀜 → 반영 대상을 우측 콘텐츠로 한정
- 라이트 모드 배경이 칙칙함 → 배경 명도 상향, 팝업/모달 라이트 디자인 전환
- 컴포넌트 시트의 추측 컴포넌트가 실제 화면과 대비·형태가 상이 → 동일 기준으로 재검증
- 라이트 모드에서 HTML 내보내기 버튼이 흰색으로 비가시 → 모드별 버튼 색 정정
