## 개요

v0.8.0 이후 WEMB Studio의 **브랜드 아이덴티티·내비게이션 정보구조(IA)**를 정비하는 한편, 대시보드/디지털 트윈에 **직접 편집(인터랙션) 역량**과 **디자인 토큰 기반 조정 컨트롤**을 신규 도입함. 링크 공유용 **OG 카드·파비콘** 자산을 구축하고 좌측 내비게이션을 Figma 기준으로 **전면 리디자인**했으며, 기준안·리디자인안·색감안 **A/B/C 3종 병행 관리 체계**를 수립함. 아울러 디지털 트윈 화면에 **인라인 편집(패널 자유 배치 + 텍스트 직접 수정)** 기능을 구현하고, **디자인 편집 컨트롤 체계를 토큰 슬라이더로 확장**하며, **사이드바 시각 위계·접근성**을 개선함.

## 핵심 기능 요약

- **브랜드 공유 카드:** SVG 파비콘과 OG 이미지를 제작하고 전 산출물에 OG·Twitter 카드 메타를 적용해, 메신저·SNS 링크 공유 시 브랜드 일관성 있는 미리보기 카드가 노출되도록 확보
- **Figma식 내비게이션 IA 리디자인:** 상단 탭 방식을 폐기하고 얇은 아이콘 레일 + 드롭 아코디언 드로어로 재설계해, 옵션 메뉴가 우측 프리뷰를 밀어내지 않고 버튼 아래로 인플레이스(in-place) 확장
- **A/B/C 병행 관리 체계:** 기준안·리디자인안·색감안을 독립 산출물로 분기해 이해관계자 비교·방향성 의사결정을 지원하고, 최종적으로 정본을 단일화하는 형상관리 프로세스 수립
- **디지털 트윈 인라인 편집:** '대시보드 편집'을 디지털 트윈 화면까지 확장 — Pointer Events 기반 패널 자유 배치(transform)와 `contenteditable` 텍스트 직접 수정을 제공하고, 편집 상태를 localStorage에 영속화·복원. 편집 중 원 기능(뷰 전환·트리·필터) 간섭은 이벤트 캡처로 차단
- **디자인 토큰 컨트롤 확장:** 패널 간격·내부 여백·그림자(입체감)·테두리·표면 강도를 CSS 커스텀 프로퍼티 슬라이더로 토큰화하고 밀도 프리셋(콤팩트/보통/넓게)을 추가해, 대시보드·디지털 트윈 양 화면에 일괄 반영

## 기술 구성

- **구조:** 단일 HTML + CSS 커스텀 프로퍼티 토큰 + Vanilla JS(외부 의존성 없음), 툴 크롬은 고정 팔레트 / 생성 테마·편집은 프리뷰 레이어에만 적용하는 관심사 분리 유지
- **인라인 편집:** Pointer Events 기반 드래그 리포지셔닝(`transform: translate` + 경계 클램프), `contenteditable` 텍스트 편집, 화면별 편집 컨트롤러 분리(가드) 및 화면 전환 시 상태 정리
- **디자인 토큰:** `--panel-gap`·`--pad-scale`·`--panel-shadow`·`--panel-border-w`·`--panel-bg` 등 CSS 변수 슬라이더로 노출, `.panel`/`.cols`/`.stage`와 `.dt-*` 양쪽에 바인딩(디지털 트윈은 유리감 유지 위해 반투명 합성)
- **영속화:** `localStorage`(`wemb-dt-layout`·`wemb-dt-content`·`wemb-dash-*`)에 저장·복원, 전체 초기화가 신규 키까지 일괄 커버하도록 보강
- **접근성:** 2차 텍스트 대비비 WCAG AA→AAA 수준 상향, `aria-disabled`·서술형 툴팁 표준화, 이중언어 라벨 표기 일원화

## 이슈 & 해결

- 사이드바 접힘 시 좌상단 고정 햄버거와 디지털 트윈 헤더 로고가 겹침 → 접힘 상태 전용 좌측 패딩 규칙을 추가해 회피
- 세그먼트 활성(볼드) 시 'Digital Twin' 라벨 2줄 줄바꿈, 상단 탭 부제 폰트 환경별 줄바꿈(Pretendard 렌더링 폭) → `white-space: nowrap` + 폭·자간 최적화로 한 줄 보장
- 표면 강도 조절이 시각적으로 반영되지 않음(`--bg-surface`↔`--bg-page` 근접색 + `transition` 측정 착시) → 양방향 흰색 틴트 방식으로 재설계해 가시적 대비 확보
- 디자인 컨트롤이 디지털 트윈에 미적용(DT 카드가 별도 변수 사용) → `--panel-*` 토큰을 DT 요소에 연결하되 반투명(유리) 특성 보존
- 리셋 컨트롤 2종이 동일 형태로 중복 인지 → 위계 재정비(섹션 전용 subtle vs 주 리셋) 및 중복 버튼 제거, 전체 초기화로 기능 일원화
