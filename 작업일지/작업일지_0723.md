## 개요

WEMB Studio의 **테마 토큰 아키텍처**와 **디자인 시각 규칙**을 전면 점검·정비함. 라이트 모드가 프리뷰에 반영되지 않던 **토큰 캐스케이드 구조 결함**을 근본 수정하고, 화면·버전·명암 모드 **8개 조합에 대한 대비 전수 감사 체계**를 구축해 기준 미달 항목을 전량 해소함. 3D 배경 위 UI 요소의 판독성을 재설계하고 **글라스모피즘 표면 규격을 단일 토큰으로 통합**했으며, 라운드 요소의 **단면 보더를 사방 링 방식으로 표준화**함. 아울러 **타이틀 정보 위계 3단계 표준**을 정립하고 중첩 컨테이너를 해소해 사이드바 정보 밀도를 개선했으며, 대비 검증표에 **미달 항목 → 편집 지점 이동 인터랙션**을 신규 구현함.

## 핵심 기능 요약

- **토큰 캐스케이드 구조 재설계:** 패널 표면색이 최상위 루트에서 색상으로 연산되어, 툴 크롬에 고정된 다크 값으로 확정된 채 하위로 상속되던 구조를 발견. CSS 커스텀 프로퍼티의 `var()` 치환이 **선언 요소 기준으로 이루어지는** 특성에 따라, 연산 시점을 팔레트를 보유한 프리뷰 컨테이너로 이관하고 루트에는 비율값만 전달하도록 재설계
- **대비 전수 감사 체계:** 반투명 레이어 합성과 배경 이미지 픽셀 샘플링까지 반영해 실효 배경색을 산출하는 측정 하니스를 구축하고, 8개 조합 전 요소를 자동 검수. 역할색을 전경(글자·테두리)으로 사용할 때의 명도 클램프 토큰 체계를 도입해 시드 컬러가 바뀌어도 대비가 유지되도록 보장
- **배경 이미지 위 판독성 확보:** 테마가 적용되지 않는 고정 배경 자산 특성상 뷰포트 크기에 따라 라벨 배후 밝기가 변동하는 문제를 뷰포트 스윕으로 계량화하고, 모드 비종속 고정 색상 + 다층 스크림 방식으로 재설계. 산재한 유리 표면 불투명도를 단일 토큰으로 통합하고 다크 모드 임계값을 재산정
- **보더 시각 규칙 표준화:** 라운드 요소에 단면 보더를 적용할 때 모서리를 따라 보더가 꺾이는 현상을 전수 조사·제거하고, 선택·활성 상태 표현을 사방 1px 링으로 일원화. 구분선 성격의 단면 보더는 성격이 다르므로 존치 대상으로 분류
- **검증 → 수정 플로우 연결:** 대비 검증표에서 미달 항목을 클릭하면 사이드바 접힘·아코디언 접힘을 순차 해제하며 해당 색상 편집 지점으로 이동하고, 그 색이 걸린 검사 항목과 현재 수치를 실시간 갱신되는 진단 카드로 제시

## 기술 구성

- **토큰 스코프:** 커스텀 프로퍼티 값 내부의 `var()`는 선언 요소에서 치환·확정된다는 원칙에 따라, 색상 연산은 팔레트 보유 컨테이너(`.main`·`.abstage`)에서 수행하고 루트에는 단위 없는 비율값만 배치
- **명도 클램프:** `oklch(from <color> ...)` 상대 색상 문법과 `min()`/`max()`를 조합해 색상(hue)·채도(chroma)는 보존하고 명도만 제한, 이미 기준을 만족하는 팔레트는 무보정 통과하도록 설계
- **표면 합성:** `color-mix` 다단 중첩으로 표면 밝기·틴트를 파생, 유리 표면 불투명도는 단일 토큰(`--dt-glass-op`)으로 통합해 모드별 임계값만 재정의
- **상태 표시:** `inset box-shadow` 사방 링 방식 채택, 테이블 행은 셀 단위로 상·하·좌·우를 분담 렌더하여 단일 링으로 결합
- **검증 하니스:** Canvas 기반 색상 파싱·레이어 합성·배경 이미지 영역 평균 샘플링으로 실효 배경을 산출, 대비 계산 후 폰트 크기·굵기별 기준(4.5:1 / 3:1)을 자동 적용

## 이슈 & 해결

- 패널 표면색이 라이트 모드에서 다크 값으로 고정 표시 → 루트 선언 시 `var()`가 툴 크롬 값으로 확정되는 구조가 원인. 연산 시점을 프리뷰 컨테이너로 이관하고 루트는 비율값만 전달하도록 분리
- 강조·상태색을 전경으로 사용 시 배경 대비 보정 장치 부재(배지·태그는 동일 색상 옅은 틴트 위에 얹혀 기준 대비가 추가 감소) → 명도 클램프 토큰 체계 도입으로 일괄 해소
- 3D 배경 위 라벨이 뷰포트 크기에 따라 판독 불가 수준까지 저하 → 색상 단독으로는 보장 불가함을 스윕 측정으로 확인, 스크림 방식으로 전환
- 다크 모드에서 배경 이미지의 밝은 영역이 유리 표면을 투과해 텍스트 대비 저하 → 라이트 모드는 기존 불투명도 유지, 다크 모드만 임계값 상향
- 라운드 요소의 단면 보더가 모서리에서 꺾여 시각적 이물감 발생, 슬라이딩 인디케이터와 실제 버튼 좌표 불일치(그리드 트랙이 최소 콘텐츠 폭으로 확장) → 사방 링 전환 및 균등 트랙 강제로 정렬 일치
- 아코디언 헤더 스타일 규칙이 두 곳에 중복 정의되어 후순위 규칙이 실제 적용 → 위계 값을 단일 지점으로 통합
- 컨테이너 중첩 해소 과정에서 헤더 내 도움말 어포던스 소실 위험 → 섹션 타이틀로 이관하고 선택된 방식에 해당하는 항목만 노출되도록 연결
