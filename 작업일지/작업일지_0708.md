## 개요

WEMB Studio(컬러 테마 스튜디오)를 **비디자이너 PM도 안전하게 쓸 수 있는 도구**로 끌어올리는 개선을 진행함. 검증 모달을 열지 않아도 위험을 알려주는 **실시간 대비 안전망**과 첫 실행 **온보딩·스포트라이트 투어**를 도입하고, 새로고침에도 사라지지 않는 **작업 자동 저장**과 **스냅샷 파일 백업**으로 데이터 손실을 차단함. 색 선택 방식(직접색·사진·추천)을 **세그먼트 탭**으로 통합하고, Enterprise 버전이 스냅샷·공유·내보내기에서 유실되던 **기능 격차를 수정**함. 마지막으로 UI/UX 크리틱(33/40) 결과를 반영해 **접근성·대비 검증 모델·컴포넌트 일관성·정보 밀도**를 전면 정비하고(v0.7.0), 형상관리에 반영함.

## 핵심 기능 요약

- **자동 대비 안전망:** 검증표를 열지 않아도 글자·색 대비 미달을 실시간 배너로 경고하고, 한 번의 클릭으로 자동 보정(되돌리기 지원). 내보내기 창을 열 때도 미달 시 재차 가드
- **한눈에 만드는 색 소스 탭:** 직접색·사진·추천을 상단 세그먼트로 통합해, 지금 쓰는 방식의 메뉴만 노출(나머지 접힘)함으로써 "어느 것이 적용 중인지" 혼란 제거
- **잃지 않는 작업:** 작업 중 테마를 자동 저장해 새로고침·재접속에도 복원, 스냅샷은 파일(.json)로 백업/복원 가능해 브라우저 초기화 시에도 보존
- **정합성 있는 내보내기:** Enterprise 확장 토큰까지 포함하고, 같은 기준색으로 다크+라이트를 한 번에 생성(CSS `prefers-color-scheme` / Figma 2모드), 팔레트+대비 검증표를 담은 PDF 사양서로 핸드오프
- **온보딩·둘러보기:** 첫 실행 3단계 안내 후 실제 사이드바·프리뷰·내보내기 영역을 순서대로 짚는 스포트라이트 투어 제공, 이후 '둘러보기·도움말' 버튼으로 언제든 재실행

## 기술 구성

- **구조:** 단일 HTML + CSS 변수 토큰 + Vanilla JS(외부 의존성 없음), 툴 크롬은 고정 중립 모노톤 / 생성 테마는 `.main` 프리뷰에만 적용
- **대비 계산:** WCAG 2.1 상대휘도 기반. 강조/경고 칩의 글자색을 흰/검 중 대비가 높은 쪽으로 자동 선택(`--on-point` 등)하고, 생성기·검증표·자동보정을 "자동 글자색 ≥4.5:1" 단일 기준으로 통일
- **상태 보존:** `localStorage` 키 분리(`wemb-current-theme` 자동저장 / `wemb-theme-snapshots` 스냅샷 / `wemb-onboarded`), 저장 실패(용량 초과·시크릿 모드)는 토스트로 안내
- **버전 보존:** `captureTheme()`에 `version` 포함, 공통 `themeCoreMap()`이 Flat/Enterprise 코어색을 산출해 프리뷰·A/B·스냅샷·내보내기가 동일 값 사용
- **정보 구조:** 검증·저장·내보내기를 `details`/`summary` 스텝 카드로 접고, 접근성을 위해 사이드바 스텝을 시각 순서대로 물리 배치(CSS `order` 제거) + 모달 포커스 트랩·복귀
- **온보딩 투어:** 대상 요소 좌표를 동기 계산해 스포트라이트 배치(비활성 탭 대비 폴백 포함), 접힌 스텝 카드는 자동으로 펼침

## 이슈 & 해결

- 다크 배경에서 "흰 글자 4.5:1 + 배경 4.5:1" 동시 충족이 수학적으로 불가 → 흰색 고정 대신 **자동 글자색(흰/검)** 기준으로 전환해 기본 테마가 전 항목 통과
- Enterprise가 화면에는 확장 토큰(~50색)으로 렌더되는데 내보내기·스냅샷은 10색만 반영 → `themeCoreMap()` 공유 + `version` 보존으로 화면과 산출물 일치
- 사이드바 스텝을 CSS `order`로 재배열해 Tab/스크린리더 순서가 1→4→2→3로 튐 → 마크업을 시각 순서대로 물리 재배치
- 모달을 코드로 닫을 때 프로그램적 `focus()`가 `focusin`을 누락시켜 포커스 복귀 실패 → `pointerdown` 캡처로 연 요소를 확보
- 배경 탭에서 `requestAnimationFrame` 정지로 스포트라이트가 표시되지 않음 → rAF 의존을 제거하고 동기 계산으로 전환
- 검증 모델이 소형(11px) 칩을 큰글씨(3:1) 기준으로 판정하던 맹점 → 본문(4.5:1) 기준으로 재분류
