## 개요

접근성 **검증 UX**를 다듬고, 컴포넌트 시트를 실용적인 **컴포넌트 갤러리**로 재정의함. 라이트 모드 색상 안정화를 진행함.

## 주요 작업 내역

| 구분          | 작업 내용                                                                                                  | 상태 |
| ------------- | ---------------------------------------------------------------------------------------------------------- | ---- |
| 검증표 UX     | 대비비 검증표 가독성 개선, 팝업 → 모달로 전환(디자인 유지)                                                 | 완료 |
| 자동 보정     | 자동 보정 적용 + 되돌리기, '고치기' 버튼 연결, 주의/수정 대상 항목 명확 표시                               | 완료 |
| 컴포넌트 시트 | 명도 보정 섹션 제거 후 순수 컴포넌트 갤러리로 재정의 — 화면 내 컴포넌트 + 미노출 추정 컴포넌트·상태값 포함 | 완료 |
| 버그 수정     | 라이트 버전에서 글자가 흰색으로 묻히는 문제 수정, 알림(Alerts) 텍스트 길이 오류 확인                       | 완료 |

## 핵심 기능 요약

- **검증 편의:** 대비비 검증을 모달로 명확화하고 무엇을·어떻게 고쳤는지 표시
- **컴포넌트 카탈로그:** 화면에 없는 상태값까지 한 곳에서 열람

## 기술 구성

- **구조:** 단일 HTML + CSS 변수 토큰 + Vanilla JS
- **모드:** `html[data-mode]` 기준 라이트/다크 텍스트 토큰 정합

## 이슈 & 해결

- 라이트 모드에서 텍스트가 흰색으로 표시 → 모드별 텍스트 토큰 정정
- 컴포넌트 시트 활용도 낮음 → '명도 보정' 도구 성격을 걷어내고 컴포넌트/상태 갤러리 용도로 재정의
- 알림 컴포넌트 텍스트 길이 초과 → 레이아웃 확인·정리
